Explanation

This animation demonstrates the use of double hashing, a technique used to resolve collisions in open-addressed hash tables. The above two hash functions are used. Both hash functions are used each time a value is inputted into the table, in order to find where to insert those values into the table. The slot in which they are hashed into follow the double hashing function, which is (The first hash function + i * the second hash function) mod 16. The variable i is intially set to 0, but every time an inputted value encounters a collision, which is when there is already a value in the hashed addressed, i keeps incrementing and the double hashing function is recomputed until the inputted value finds an empty slot in the table. When inserting a value into the table, a successful insertion is denoted by a green cell. If there is a collision, the slot where that value should have hashed to will be denoted by one or more red gradient cells, and its successful placement into the table will be denoted by the green cell. The darkest shade of red is the original slot where the value should have hashed to, and each lighter shade of red is the next slot it should have hashed to and so on.
